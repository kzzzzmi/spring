Session
 - 클라이언트가 서버에 요청(GET)
 - 서버가 클라이언트에게 응답 -> Session key 발급(출입증)
 - 이후 클라이언트는 session key도 들고 요청함
 - 서버는 처음왔던 클라이언트가 아니란걸 알 수 있음
 - 클라이언트의 프로세서가 종료되면 세션키는 없어짐(브라우저 종료될 시)
 - JSESSIONID -> Session
 - /study 안의 어떤 페이지를 가든 JSESSION은 있어서 데이터 공유 가능


Ajax
 - 비동기 처리방식 (병렬적 태스크 수행)
 1. 기존 소스코드는 순서대로 실행이 됨, 중간에 데이터를 불러와야 하는 경우가 생김, 불러오는 동안 밑의 소스는 실행 안됨
 2. 처리가 걸리거나 부분적으로 실행해야 한다면 ajax를 이용해서 병렬 테스크 수행 가능
 3. 부분적으로 실행됨
 ex) 아이디 중복확인 -> post요청 -> 현재 처리 후 response 후 view resolver가 처리하는 방식
        -> 페이지가 깜빡거림 -> ajax를 이용하고 자바스크립트 제이쿼리를 이용하여 데이터만 받아오는 방식
